# AI 写作系统 · 总纲

> 作者：wanghaiyu｜架构图：`architecture.drawio`

---

## 系统概述

本系统是一套基于多级 Agent 协作的 **AI 智能写作平台**，以「数据驱动内容生产」为核心理念，通过结构化的知识来源、双层 Agent 执行体系与工程脚本串联，实现从客户原始材料到高质量成稿的全链路自动化写作。

```
客户材料 ──▶ 产品Agent（PRD）──▶ 写作Agent（成稿）
                 ▲                     ▲
           藏经阁知识库 ─── 证据补充 ──┘
                 │
           工程脚本统一串联
```

---

## 架构分层

### 一、数据来源层 📚

知识来源为内部**藏经阁**知识库，经过以下流水线处理后对上层 Agent 提供结构化知识支撑：

| 步骤 | 说明 |
|------|------|
| **标签简化** | 对藏经阁原始标签进行精简与规范化处理 |
| **KV 索引构建** | Key = 标签打标，Value = 对应源文章；支持快速标签检索 |
| **标签检索** | 根据任务意图进行关键词/标签匹配检索 |
| **证据提取 Skill** | 对检索结果进行精细化证据抽取，输出本次任务所需的结构化证据 |

---

### 二、执行 Agent 层 🤖

执行层由两个串行的 Agent 构成，**均使用 Speckit 流程驱动**。

#### 2.1 产品 Agent 🏭

- **输入**：客户原始材料（PPT / PDF / DOCX 等多种格式）
- **输出**：PRD（产品需求文档）
- **内部流程**：调用 Speckit 流程，依次执行以下 Skills：

| Skill | 职责 |
|-------|------|
| 📥 输入结构化 Skill | 将 PPT / PDF / DOCX 等格式转换为结构化 Markdown |
| 🔗 证据补充 Skill | 与数据来源层通信，将相关知识证据注入 PRD 草稿 |
| ✅ PRD 审核 Skill | 创建**产品审核子 Agent**，对 PRD 进行质量审核 |

#### 2.2 写作 Agent ✍️

- **输入**：产品 Agent 输出的 PRD
- **输出**：最终成稿
- **内部流程**：调用 Speckit 流程，依次执行以下 Skills：

| Skill | 职责 |
|-------|------|
| 🖊️ 写作 Skills（已有） | 调用多种风格/类型的写作 Skill 进行内容生产 |
| 📝 成稿审核 Skill | 创建**写作审核子 Agent**，对成稿进行质量审核 |

---

### 三、审核 Agent 层 🔎

审核层由执行 Agent 的审核 Skill 动态创建，作为**子 Agent** 异步运行，保证产出物质量。

| 子 Agent | 触发方 | 审核内容 |
|----------|--------|----------|
| 🏭 产品审核 Agent | PRD 审核 Skill | PRD 的完整性、逻辑性、需求覆盖度 |
| ✍️ 写作审核 Agent | 成稿审核 Skill | 成稿的可读性、准确性、风格一致性 |

> 审核 Agent 发现问题后可回写修改意见，触发对应执行 Agent 的迭代修正。

---

### 四、工程脚本串联层 🔧

通过统一的**工程脚本**对整体流程进行编排与管理：

- 串联 产品 Agent → 写作 Agent 的顺序执行
- 管理 数据来源层 ↔ 执行 Agent 层 之间的通信
- 负责任务状态追踪、错误重试与日志记录

## 完整执行流程

```
┌──────────────────────────────────────────────────────────┐
│                     工程脚本 (Orchestrator)               │
└───────────────────────┬──────────────────────────────────┘
                        │
          ┌─────────────▼─────────────┐
          │        客户材料输入        │
          │  (PPT / PDF / DOCX / ...)  │
          └─────────────┬─────────────┘
                        │
          ┌─────────────▼─────────────┐        ┌─────────────────────┐
          │        产品 Agent          │◀───────│   藏经阁 知识库      │
          │  Speckit 流程              │  证据   │  标签检索 + 证据提取 │
          │  ① 输入结构化 Skill        │        └─────────────────────┘
          │  ② 证据补充 Skill          │
          │  ③ PRD 审核 Skill          │──── 创建 ──▶ 产品审核 Agent
          └─────────────┬─────────────┘
                        │ PRD
          ┌─────────────▼─────────────┐
          │        写作 Agent          │
          │  Speckit 流程              │
          │  ① 写作 Skills             │
          │  ② 成稿审核 Skill          │──── 创建 ──▶ 写作审核 Agent
          └─────────────┬─────────────┘
                        │
          ┌─────────────▼─────────────┐
          │          最终成稿           │
          └───────────────────────────┘
```

## 关键设计原则

1. **数据驱动**：所有内容生产均有来自藏经阁的结构化证据支撑，避免 AI 幻觉
2. **两级 Agent**：执行 Agent 负责生产，审核 Agent 负责质量把关，职责分离
3. **Speckit 标准化**：所有 Agent 内部流程统一使用 Speckit 驱动，保证可维护性与可扩展性
4. **Skill 模块化**：各能力以 Skill 形式封装，便于独立迭代与复用
5. **工程脚本解耦**：Agent 之间通过工程脚本串联，而非直接依赖，保持松耦合
